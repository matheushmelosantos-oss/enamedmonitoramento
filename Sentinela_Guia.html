Fase 1
Base de Dados
Dias 1–4 · Planilha mestre, INDECX, Medway e dados da coordenação
Dia 1
📋
Criar a planilha mestre dos alunos
Segunda · ~2h · Google Sheets
▼
Por que começa aqui? O Sentinela cruza 4 fontes de dados — INDECX, Medway, simulado e presença. Para isso funcionar, precisa de uma planilha central que receba tudo e calcule o score composto. Esta planilha é o único ativo que fica com a Mandic independente de qualquer sistema.
✓
Crie uma planilha no Google Drive chamada Sentinela_BancoMestre
drive.google.com → + Novo → Planilhas Google
Sheets
✓
Crie 5 abas dentro da planilha
ALUNOS · PESQUISAS · ACADEMICO · MEDWAY · RESPONSAVEIS

✓
Na aba ALUNOS, cole os cabeçalhos abaixo na célula A1
Sheets
Cabeçalhos aba ALUNOS — cole em A1
matricula nome_aluno email unidade turma coordenador simulado_1 simulado_2 simulado_3 simulado_media presenca_pct medway_conclusao_pct medway_logins_30d ultimo_nps ultimo_csat_prep ultimo_csat_semanal ultimo_csat_mensal score_composto nivel_risco flag_silencio data_atualizacao
✓
Na aba PESQUISAS, cole os cabeçalhos para as respostas do INDECX
Sheets
Cabeçalhos aba PESQUISAS — cole em A1

✓
Na aba RESPONSAVEIS, crie 4 colunas: unidade, cx_responsavel, coord_academico, apoio_psicopedagogico
Preencha com os emails de cada responsável por unidade. O Make vai usar isso para saber para quem enviar os alertas conforme o protocolo.

✅
Dia 1 concluído quando: Planilha criada com 5 abas e todos os cabeçalhos no lugar.
Dia 2
🔑
Configurar o INDECX e verificar as pesquisas
Terça · ~1h · INDECX
▼
Ponto crítico: O INDECX tem 4 pesquisas ativas para o Pré-ENAMED. Cada uma precisa ter a matrícula do aluno como indicador — sem ela, não dá para cruzar as respostas com o cadastro do aluno e calcular o score.
✓
Acesse app-indecx.com → Configurações → Dados da Conta → copie a company-key
Guarde em local seguro. Não compartilhe por WhatsApp ou email.
INDECX
✓
Anote os controlIds das 4 pesquisas ativas do Pré-ENAMED
Vá em Ações e anote: CSAT Aulas Preparatórias · CSAT Internato Semanal · CSAT Internato Mensal · NPS Ciclo 26.1
INDECX
✓
Confirme que o campo "matricula" existe como indicador em TODAS as 4 pesquisas
Entre em cada ação → Indicadores. Se não existir, crie agora com o nome exato "matricula" (minúsculo, sem acento). Este é o campo que liga pesquisa → aluno → score.
INDECX
✓
Teste a API via Swagger do INDECX
Acesse indecx.com/api-docs → authorize com a company-key → GET /v3/integrations/answers-info/all?limit=3 → se retornar JSON com respostas, está OK.
INDECX
⚠️
Se a matrícula não estiver como indicador: As respostas chegam no Sheets mas ficam desvinculadas do aluno — o score composto não consegue calcular. Resolva isso antes do Dia 4.
Dia 3
📥
Importar cadastro de alunos e dados acadêmicos
Quarta · ~2h · Sheets + Coordenação
▼
Cadência mensal: Segundo o protocolo do Sentinela, a coordenação acadêmica envia a planilha de simulado e presença até o dia 15 de cada mês. Este dia 3 é a primeira carga — você vai popular a aba ALUNOS e a aba ACADEMICO pela primeira vez.
✓
Peça à coordenação acadêmica a planilha com os 200 alunos da turma ENAMED 2026
Campos mínimos: matrícula, nome, email, unidade, turma. Essa lista vai para a aba ALUNOS.
✓
Cole os dados na aba ALUNOS a partir da linha 2, mantendo os cabeçalhos do Dia 1
Um aluno por linha. A matrícula precisa ser idêntica à que está cadastrada no INDECX e na Medway — sem espaços, zeros extras ou diferenças de formato.
Sheets
✓
Solicite os resultados dos simulados realizados até agora e cole na aba ACADEMICO
Colunas: matricula · simulado_1 · simulado_2 · simulado_3 · presenca_pct. Deixe em branco o que ainda não aconteceu.
Sheets
✓
Na aba ALUNOS, use PROCV para puxar os dados acadêmicos automaticamente

Fase 2
Score e Alertas
Dias 5–7 · Score composto, classificação IA e alertas por protocolo
Dia 5
🧮
Criar o score composto por aluno
Segunda · ~2h · Sheets
▼
O score é o coração do Sentinela. Combina 4 dimensões com os pesos definidos no protocolo: Performance acadêmica (30%), Engajamento (25%), Experiência/CSATs (20%) e Voz/NPS+sentimento (25%). Score 0–100 — quanto maior, maior o risco.
✓
Na coluna R (score_composto) da aba ALUNOS, crie a fórmula do score
Sheets
Fórmula do score composto — cole em R2
=SEERRO( ((1-(G2/100))*30) + ((1-((K2/100+L2/100)/2))*25) + ((1-((O2+P2+Q2)/15))*20) + ((SE(N2<=6,1,SE(N2<=8,0.5,0))*25)) , 0)
✓
Na coluna S (nivel_risco), crie a classificação automática pelos limiares do protocolo
Sheets
Fórmula nivel_risco — cole em S2 (limiares: Verde≤25, Amarelo≤50, Laranja≤75, Vermelho>75)
=SE(R2>75,"Vermelho",SE(R2>50,"Laranja",SE(R2>25,"Amarelo","Verde")))
✓
Aplique formatação condicional na coluna S para colorir automaticamente
Formatar → Formatação condicional → por texto: "Vermelho" = fundo vermelho, "Laranja" = fundo laranja, "Amarelo" = fundo amarelo, "Verde" = fundo verde.
Sheets
✓
Arraste todas as fórmulas para as 200 linhas e verifique se os scores fazem sentido
Abra os perfis de alunos que você sabe que estão em risco — o score deles deve aparecer como Laranja ou Vermelho. Se não estiver, revise o mapeamento dos campos.
Sheets
Dia 6
🚨
Configurar alertas automáticos por protocolo
Terça · ~2h · Make + Claude API
▼
O protocolo do Sentinela define: Vermelho = comitê em 48h (CX + Coord. Acadêmica + Apoio Psicopedagógico). Laranja = conversa individual em 7 dias. O Make detecta, a Claude classifica o motivo e aciona a pessoa certa automaticamente.
✓
Crie conta na Anthropic e obtenha a chave da Claude API
console.anthropic.com → crie conta → API Keys → Create Key → adicione $5 de crédito em Billing.
Claude
✓
Crie um novo cenário no Make chamado "Sentinela — Triagem de Risco"
Este cenário roda toda vez que uma nova resposta chega. Adiciona o Router após o Iterator do Dia 4.
Make
✓
Configure o Router com 2 rotas: score > 50 (laranja/vermelho) e score ≤ 50 (verde/amarelo)
Make
✓
Na rota de risco, adicione HTTP para chamar a Claude API
URL: https://api.anthropic.com/v1/messages · Headers: x-api-key, anthropic-version: 2023-06-01
Claude
✓
Configure o prompt da Claude para classificar o motivo e recomendar o responsável
Claude
Prompt para a Claude API — cole no body do HTTP

Responda SOMENTE em JSON sem texto extra: {\"motivo\": \"Academico ou Operacional ou Emocional ou Relacionamento\", \"urgencia\": \"Vermelho ou Laranja ou Amarelo\", \"responsavel\": \"CX ou Coordenacao ou Psicopedagogia ou Comite48h\", \"resumo\": \"uma frase do que está acontecendo\", \"acao\": \"uma frase do que fazer agora\"}. ✓
Adicione Gmail → Send Email para enviar o alerta ao responsável correto
Vermelho → email para CX + Coord. Acadêmica + Psicopedagogia (comitê 48h). Laranja → só para CX. Use a aba RESPONSAVEIS para buscar os emails pela unidade do aluno.
Make
✓

Dia 7
📧
Criar a triagem semanal do CX
Quarta · ~1h · Make
▼
Protocolo semanal do Sentinela: CX faz triagem dos vermelhos e laranjas toda semana + atualização das tratativas em aberto. Este cenário gera automaticamente o resumo da semana toda segunda-feira às 08h.
✓
Crie um cenário "Sentinela — Triagem Semanal CX"
Make
✓
Configure para rodar toda segunda-feira às 10h55 UTC (= 07h55 BRT)
Make
✓
Sheets → busca alunos com nivel_risco = Vermelho ou Laranja → Claude gera resumo → email para o time CX
O email deve conter: lista de vermelhos e laranjas, quantos têm tratativa aberta, aluno mais urgente (menor score), e link para o painel Sentinela.

✓
Agende revisão do sistema em 30 dias para avaliar critérios de sucesso do protocolo
Metas do protocolo: 100% dos vermelhos com tratativa em até 48h · latência máxima dado→painel ≤30 dias · ≥80% das tratativas resolvidas no fechamento do ciclo.
