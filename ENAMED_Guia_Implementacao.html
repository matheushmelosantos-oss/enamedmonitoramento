Dia 2
🔑
Configurar o INDECX e testar a API
Terça · ~1h · INDECX
▼
O que é isso? O INDECX tem pesquisas específicas para o Pré-ENAMED — NPS, CSAT de preparatórias, CSAT de internato. Você precisa identificar quais são, anotar os IDs e confirmar que a matrícula do aluno está sendo enviada como indicador em cada pesquisa. Sem a matrícula, não dá para cruzar com os dados da TOTVS.
✓
Acesse app-indecx.com e vá em Configurações → Dados da Conta
Copie a company-key e guarde em local seguro.
INDECX
✓
Liste todas as pesquisas ativas relacionadas ao Pré-ENAMED
Vá em Ações e anote o controlId de cada pesquisa: NPS geral, CSAT preparatórias, CSAT internato semanal, CSAT internato mensal.
INDECX
✓
Confirme se a matrícula do aluno está sendo enviada como indicador em cada pesquisa
Entre em cada ação → Indicadores. Precisa existir um indicador chamado "matricula" (ou similar). Se não existir, crie agora — é a chave que vai ligar o dado da pesquisa ao dado da TOTVS.
INDECX
✓
Teste a API do INDECX para confirmar que está funcionando
Acesse indecx.com/api-docs → authorize com a company-key → teste o endpoint GET /v3/integrations/answers-info/all?limit=3. Se retornar respostas, está OK.
INDECX
⚠️
Ponto crítico: Se a matrícula não estiver como indicador nas pesquisas, o cruzamento com a TOTVS não vai funcionar. Resolva isso antes de prosseguir.
Dia 3
📥
Preparar e importar os dados da TOTVS
Quarta · ~2h · TOTVS + Sheets
▼
O que é isso? A TOTVS vai exportar os dados acadêmicos como planilha (notas de simulado, presença, status de matrícula). Antes de importar, precisa limpar e padronizar — coluna de matrícula no mesmo formato dos dois sistemas, sem espaços ou caracteres estranhos.
✓
Exporte o relatório de alunos da TOTVS com os campos necessários
Campos mínimos: matrícula, nome, unidade, turma, notas dos últimos 3 simulados, percentual de presença. Peça para o time acadêmico se não souber como exportar.
TOTVS
✓
Abra a planilha exportada e padronize a coluna de matrícula
A matrícula precisa ser idêntica nos dois sistemas — INDECX e TOTVS. Verifique: tem zeros à esquerda? Tem letras? Remova espaços com a fórmula =TRIM(A2). Padronize tudo.
Sheets
✓
Cole os dados na aba ACADEMICO da planilha ENAMED_BancoMestre
Copie os dados da planilha da TOTVS e cole na aba ACADEMICO a partir da linha 2. Mantenha os cabeçalhos que você criou no Dia 1.
Sheets
✓
Na aba ALUNOS, cole a lista de alunos com matrícula, nome, unidade e turma
Esta aba é o cadastro mestre. Cada aluno aparece uma vez. Use a mesma lista da TOTVS como base.
Sheets
💡
Dica: Combine a fórmula PROCV (ou XLOOKUP) para puxar os dados da aba ACADEMICO para a aba ALUNOS usando a matrícula como chave. Exemplo: =PROCV(A2,ACADEMICO!A:L,8,0) puxa a média de simulados.

Fase 2
Score Composto e Alertas
Dias 5–7 · Cálculo do score, classificação por IA e alertas automáticos
Dia 5
🧮
Criar o score composto por aluno
Segunda · ~2h · Sheets
▼
O score composto é o coração do sistema. Ele combina NPS, CSAT e dados acadêmicos em um número de 0 a 100 que indica o nível de risco de cada aluno. Quanto menor o score, maior o risco.
✓
Na aba SCORE, crie a fórmula de score composto para cada aluno
O score combina 3 dimensões com pesos iguais (33% cada): NPS normalizado (nota/10 × 100), CSAT normalizado (nota/5 × 100) e desempenho acadêmico (média simulados normalizada). Score = (NPS×0,33) + (CSAT×0,33) + (Acadêmico×0,34)
Sheets
✓
Crie a coluna nivel_risco com classificação automática
Sheets
Fórmula para a coluna nivel_risco
=SE(L2<40,"Vermelho",SE(L2<60,"Laranja",SE(L2<75,"Amarelo","Verde")))
✓
Crie a coluna flag_silencio para detectar alunos que não respondem pesquisas
Sheets
Fórmula para flag_silencio (sem resposta há mais de 60 dias)
=SE(OU(N2="";HOJE()-DATEVAL(N2)>60),"SIM","NÃO")
✓
Aplique formatação condicional na coluna nivel_risco
Vermelho = fundo vermelho · Laranja = fundo laranja · Amarelo = fundo amarelo · Verde = fundo verde. Isso cria o heatmap visual na planilha.
Sheets
Dia 6
🚨
Configurar alertas automáticos de risco
Terça · ~2h · Make + Claude API
▼
Hoje o sistema fica inteligente. Quando um aluno entra em risco vermelho ou laranja, o Make detecta, a Claude API classifica o motivo e gera um resumo, e o coordenador recebe um email de alerta com contexto completo.
✓
Crie uma conta na Anthropic e obtenha a chave da Claude API
Acesse console.anthropic.com → crie conta → API Keys → Create Key. Adicione $5 de crédito em Billing — suficiente para centenas de classificações.
Claude
✓
No Make, adicione um Router após o Iterator do cenário de ingestão
Rota A: score < 60 (risco vermelho ou laranja) → aciona alerta. Rota B: score ≥ 60 → segue sem alerta.
Make
✓
Adicione HTTP para chamar a Claude API com contexto do aluno
URL: https://api.anthropic.com/v1/messages · Headers: x-api-key, anthropic-version: 2023-06-01 · Body: monte o prompt com nome, nota NPS, feedback, média de simulados e presença
Claude
✓
Configure o prompt da Claude para classificar motivo e gerar plano de ação
Claude

Responda SOMENTE em JSON sem texto extra: {\"motivo_principal\": \"Academico ou Relacionamento ou Operacional ou Emocional\", \"nivel_urgencia\": \"Imediato ou Esta semana ou Monitorar\", \"resumo\": \"uma frase do que está acontecendo\", \"acao_recomendada\": \"uma frase do que fazer\"}. Dados: Nome: {{nome}}, NPS: {{nota}}/10, Feedback: {{feedback}}, Média simulados: {{media_simulados}}, Presença: {{presenca_pct}}%" }] }
✓

Subject: 🚨 Aluno em risco — {{nome}} · {{unidade}} · Score {{score}} · Body: inclua nome, score, nível de risco, motivo classificado pela IA e ação recomendada.
